# BarberShop Booking System (Business & System Analysis)

Проект по проектированию системы онлайн-записи для сети барбершопов. Основная цель — автоматизация бизнес-процессов, расширение клиентской базы и минимизация ручного труда при информировании клиентов.

## 🎯 Ключевые задачи проекта

### 1. Моделирование бизнес-процессов (Use Case Analysis)
*   **Проектирование сценариев:** Разработаны детальные сценарии для ключевых процессов:
    *   **UC02:** Формирование расписания мастера (управление слотами, обработка конфликтов бронирования).
    *   **UC04:** Регистрация клиента менеджером (валидация данных, генерация временных доступов).
*   **Детализация взаимодействий:** Описана последовательность шагов с разделением ответственности между клиентом, сервером и внешними сервисами.

### 2. Проектирование интеграций и API (System Design)
*   **Проектирование методов:** Описаны REST-запросы для интеграции (HTTP-методы, URL-пути, заголовки).
*   **Выбор стратегии интеграции:** Обосновано применение безопасных и идемпотентных методов для различных CRUD-операций.
*   **Работа со справочниками:** Разработаны схемы синхронизации справочников (передача дельты, поиск соответствий, полная передача).

### 3. Техническая документация и Маппинг
*   **Data Mapping:** Выполнено сопоставление данных между системой-отправителем, сообщением и системой-получателем (типы, размерность, обязательность).
*   **JSON Structure:** Спроектированы тела запросов и ответов для интеграционных методов.

### 4. Обработка исключений (Error Handling)
*   Разработано более **15 регламентов обработки ошибок** для каждого метода (логические ошибки, нарушение типов данных, недоступность внешних систем).
*   Классифицирована критичность ошибок (Warning / Critical / Blocker) и определены требования к ответам системы (Error Codes & Messages).

## 🛠 Технологический стек & Компетенции
*   **Методологии:** REST, Use Case Modeling (UML), BPMN.
*   **Инструменты проектирования:** Проектирование структур данных, маппинг полей, описание API.
*   **Архитектурные концепции:** Масштабируемость, Асинхронность, Кеширование, Идемпотентность методов.
